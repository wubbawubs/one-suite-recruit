
# OneTime Recruit — Website Rebuild Plan

## Overview
Rebuild the OneTime Recruit website as a premium, enterprise-grade recruitment site. Starting with the full Dutch (NL) website, extracting and upgrading the current branding, with the multilingual architecture (single domain, `/nl /de /en` folders) prepared from the start.

---

## Phase 1: Design System & Foundation

### Brand Token Setup
- Extract current logo and green/gold palette from the existing site
- Upgrade to a premium palette: **Deep Navy** primary, **Warm White** backgrounds, **Cool Gray** UI elements, **Copper/Orange** accent for CTAs
- Keep the OneTime Recruit tree logo as-is
- Set up a serif + sans-serif typography pairing for executive authority
- Create consistent spacing system (8px grid)

### Core Component Library
- **Header**: Sticky navigation with language switcher (NL/DE/EN), primary CTA ("Plan een gesprek"), and mobile hamburger menu
- **Footer**: Full navigation, OneGroup ecosystem links (OneRooted, OneSource, Click Larasio), legal links, social media
- **Hero variants**: Full-width split layout, centered hero with proof bar
- **Trust bar**: Client logos + key metrics (7500+ specialists, placements, etc.)
- **Service cards**: 3-column card layout for service offerings
- **Testimonial/case study cards**: Problem → approach → result format
- **CTA strips**: Call-to-action sections with booking/contact options
- **FAQ accordion**: Collapsible Q&A sections

## Phase 2: Dutch (NL) Website — All Pages

### Homepage (`/nl`)
1. Hero section: "DE nieuwe standaard voor Executive Search" with primary CTA + secondary CTA (vacancies)
2. Proof bar: Enterprise client logos (AFAS, MAS Aviation) + key metrics
3. Services overview: 3 cards — Executive Search, Leadership Advisory, Recruitment Operating Model
4. Why OneTime Recruit: 3-5 differentiators with evidence
5. How it works: Step-by-step process (intake → search → shortlist → assessment → placement → aftercare)
6. Case studies highlight: 2-3 mini case studies with measurable outcomes
7. Insights teaser: Latest articles preview
8. CTA strip: Book a call + direct contact
9. Footer with OneGroup ecosystem

### Services Pages
- **Diensten** (overview): Summary of all service offerings with links
- **Executive Search** (pillar page): Full service detail — deliverables, process, timelines, assessment methodology, FAQ
- **Leiderschap & Directie** (pillar page): C-level/Director hiring focus

### Industry Pages
- Aviation recruitment
- Tech/SaaS leadership hiring
- Finance & compliance leadership
- Template structure: Industry challenges → roles we hire → case studies → CTA

### Company Pages
- **Over ons**: Team, mission, story, values, OneGroup connection
- **Hoe we werken**: Detailed process explanation, the "new standard" model explained
- **Referenties / Case studies**: Client success stories with metrics
- **OneGroup**: Ecosystem page showing OneTime Recruit, OneRooted, OneSource, Click Larasio

### Lead Generation Pages
- **Contact**: Short form + calendar booking embed + phone/WhatsApp
- **Opdrachtgevers** (For employers): Tailored landing page for enterprise clients
- **Vacatures**: Job listing page (link to Workable or embedded)

### Insights/Blog
- Article template page with author bio, reading time, table of contents, related articles, internal linking
- A few placeholder articles to demonstrate the layout

## Design Direction
- **Premium & enterprise**: Clean, confident, lots of whitespace, high-contrast typography
- **Trust-forward**: Client logos visible early, metrics prominent, real photography placeholders
- **Conversion-focused**: Clear CTA hierarchy — primary "Plan een gesprek", secondary for vacancies/case studies
- **Mobile-first**: Fully responsive, optimized for executive-level visitors on any device
- **Dark mode support**: Optional, but the design system will support it

## What This Delivers
A complete, polished Dutch website for OneTime Recruit that positions you as THE premium executive search partner in the Netherlands, with the architecture ready to expand into German and English markets. All content is static (editable via Lovable), with the component library making future page creation fast and consistent.
